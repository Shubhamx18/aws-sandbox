# Limitations of AWS Lambda

1. Maximum execution timeout is 15 minutes per function invocation.
2. Limited memory allocation (128 MB to 10,240 MB).
3. Cannot natively run GPU workloads.
4. Execution environment is stateless.
5. Limited local storage (/tmp directory with 512 MB).
6. Limited support for WebSockets (requires API Gateway).
7. Size of deployment package: 50 MB (zip) or 10 GB (container image).
8. Debugging is more complex than traditional servers.
9. Limited control over underlying infrastructure.
10. Cold starts can affect latency.
11. Limited native support for long-running or stateful applications.
12. You must rely on other AWS services for persistent storage.
